## 5. 暂缓功能分析

### 5.1 WASM 插件系统

**暂缓原因**:
```
1. Rust WASM 宿主/插件生态复杂
2. 当前 8 个内置工具已覆盖 80% 场景
3. MCP 协议是更好的扩展方案
4. 无实际插件需求
```

**替代方案**: 优先完善 MCP Client，利用社区 5800+ MCP servers

**重新评估时机**: 项目成熟后，有明确插件需求时

---

### 5.2 多平台桥接（Telegram/飞书/Discord）

**暂缓原因**:
```
1. Gateway 已提供 HTTP API
2. 各平台 SDK 集成工作量大
3. 消息格式适配复杂
4. 需要持续的维护投入
```

**替代方案**: 
- 完善 Gateway WebSocket API
- 提供 webhook 接口
- 让社区开发各平台适配器

**重新评估时机**: Gateway 稳定，有用户明确需求

---

### 5.3 Tauri 桌面端

**暂缓原因**:
```
1. TUI 已足够验证核心交互
2. Tauri 增加构建复杂度
3. Web 界面优先级低于 API
4. 当前团队规模不适合多前端
```

**替代方案**: 完善 TUI 体验，后续可考虑 Web UI

**重新评估时机**: 核心框架稳定后
